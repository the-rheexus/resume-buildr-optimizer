# Cover Letter Voice Pass

Applies only to the Pass 3 final cover letter. The Pass 2 cover letter draft is
composed from the bank Section 5.7 component bank plus
`@knowledge/cover_letter_standards.md`, but not in Josh's voice yet — this pass
rewrites it in his voice.

## Voice source

Apply `@knowledge/writing_style_profile-josh_v2.md`'s rules and guidelines on top
of `@knowledge/cover_letter_standards.md`. The style profile adjusts voice; the
cover letter standards govern structure and length. Both apply — the style
profile does not override the structural rules.

## Never applied to the resume

`writing_style_profile-josh_v2.md` is scoped to cover letters only. It has zero
bearing on resume word choice, tone, or phrasing at any pass. Resume language is
governed by `@raw-resume-docs/master_experience_bank.md` bullet content plus
`.agents/rules/resume-format.md`.

## Voice pass ≠ content pass

Content is locked from Pass 2. Do not introduce any new factual claims in this
rewrite — only phrasing, voice, and tone change. Every claim must still trace to
a bank fact or a checkpoint-sourced willingness statement labeled as such.

## Required output subsections

Include both directly in the final cover letter file:

1. **Writing-profile self-check** — run explicitly through the style profile's own
   self-check criteria (em-dashes, semicolons, ellipses, single quotes, colons in
   prose, exclamation points, sentence fragments, contractions, additive
   connector choice, plain verbs, active voice, sentence rhythm), stating the
   count for each and marking pass/fail.
2. **Fact / rule check** — reconfirm every claim traces to a bank fact (with
   M-number or bank section reference), or is a checkpoint-sourced willingness
   statement labeled as such. Confirm nothing new was introduced.

## Salutation

Use the hiring manager's name if findable via LinkedIn, the company careers page,
the team page, or a targeted Google search. Fall back to `Dear Hiring Manager` if
genuinely unfindable — never `To Whom It May Concern`.

## Company detail

Reference at least one company-specific detail from
`@job-descriptions/{company_slug}-company_notes.md` if the free-form body has
content. If the body is `None`, source the detail from the JD (e.g., a stated
initiative or transformation model) and note that in the Fact / rule check.
